## 활동보고 제출 API

### 활동보고 공통 비즈니스 규칙

- 그룹장만 제출 가능
- 제출 시 현재보다 `endAt`이 더 나중인 `big_seminar`가 있어야 함 — 없으면 400
- 보고서(`presentation=false`) 또는 세미나 발표(`presentation=true`) 중 하나 선택
- 파일 제출 필수
- 파일은 R2에 저장하고 해당파일의 url을 요청바디에 넣어 제출
- 등록/수정/취소(삭제) 시 그룹 멤버 전체 + 운영진에게 알림

### 관련 DB 테이블

| 테이블               | 역할                                                |
| -------------------- | --------------------------------------------------- |
| `group`              | 그룹 존재 확인 + `master` 컬럼으로 그룹장 여부 확인 |
| `big_seminar`+`schedule` | 현재로부터 가장 가까운 세미나 존재 여부 확인            |
| `group_activity_report`      | 활동보고 |

### API

```
POST /group/:groupId/activity-report
```

#### 요청 바디

| 이름           | 타입      | 설명                                    |
| -------------- | --------- | --------------------------------------- |
| `isPresentation` | boolean   | false=보고서, true=세미나 발표(default) |
| `reportFile`   | R2 url | 제출 자료, 파일 하나만, 여러파일일 경우 압축파일로 |

#### 응답

```
201 Created
```

| 이름           | 타입        | 설명                      |
| -------------- | ----------- | ------------------------- |
| `id`           | 정수        | 생성된 `group_seminar.id` |
| `groupId`         | bigint      | 제출한 그룹 |
| `seminarId`         | id      | 해당하는 세미나 |
| `presentation` | boolean     | 발표 여부                 |
| `reportFile`        | string | R2 key      |
| `created_at`        | timestamp | 생성 일자      |
| `updated_at`        | timestamp | 변경 일자      |

#### 테스트 케이스

1. 그룹장이 접수 중인 기수가 있을 때 보고서 제출
   - 현재보다 `endAt`이 더 나중인 `big_seminar`가 있으면 해당 세미나에 보고서 제출, 201 반환
2. 그룹장이 접수 중인 기수가 있을 때 세미나 발표 자료 제출
   - 현재보다 `endAt`이 더 나중인 `big_seminar`가 있으면 해당 세미나에 `presentation = true`로 제출, 201 반환
3. 현재보다 `endAt`이 더 나중인 `big_seminar`가 없을 때 요청
   - 400 반환 ("접수 중인 세미나가 없습니다")
4. 그룹장이 접수중인 세미나에 제출했지만 요청바디에 파일 url이 없음
   - 400 반환, 파일(보고서/ppt) 제출 필수
5. 그룹원(비그룹장) 요청
   - 403 반환
6. 등록 성공 시
   - 그룹원 전원 경험치 +50
   - 그룹 멤버 전체 + 운영진에게 알림 발송
