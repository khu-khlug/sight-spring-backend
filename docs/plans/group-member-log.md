# 그룹 활동 로그 조회

그룹 내에서 발생한 활동 로그를 조회합니다.

## API

```
GET /groups/{groupId}/logs
```

### 쿼리 파라미터

| 이름  |  타입  | 설명                                      |
| :---- | :----: | :---------------------------------------- |
| `all` | 불리언 | `true`이면 전체 조회. 기본값은 최근 100개 |

### 요청 바디

(해당 없음)

### 응답 코드 및 응답 바디

```
200 OK
```

| 이름             |   타입    | 설명                                               |
| :--------------- | :-------: | :------------------------------------------------- |
| `logs`           | 객체 배열 | 활동 로그 목록. `createdAt` 내림차순              |
| `logs.id`        |   숫자    | 로그 ID                                            |
| `logs.memberId`  |   숫자    | 행위자 회원 ID. 시스템 행위인 경우 `0`            |
| `logs.message`   |  문자열   | 로그 메시지 (HTML)                                 |
| `logs.createdAt` |  문자열   | 발생 일시 (ISO 8601)                               |
| `hasMore`        |  불리언   | 기본 조회 시 반환되지 않은 로그가 더 있는지 여부  |

### 테스트 케이스

1. 그룹 멤버는 활동 로그를 조회할 수 있다
   - 기본값은 최근 100개를 `createdAt` 내림차순으로 반환한다
   - 기본 조회 시 로그가 100개보다 많으면 `hasMore`는 `true`다
   - `all=true`이면 전체 로그를 반환한다
   - `all=true`이면 `hasMore`는 `false`다
   - 시스템이 기록한 로그는 `memberId`가 `0`이다

2. 그룹 멤버가 아니면 403을 반환한다

3. 존재하지 않는 그룹을 조회하면 404를 반환한다
