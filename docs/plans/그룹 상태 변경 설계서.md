# 그룹 상태 

만들어진 그룹의 상태를 변경합니다. 변경된 항목에 대해 그룹 멤버에게 알림이 발송됩니다.

## API

```
PATCH /groups/{groupId} / status
```


### 경로 파라미터
| 이름   |  타입     | 설명
groupId    숫자       그룹 ID


#요청 바디
{
  "state": "closed"
}



### 상태값
progress : 진행중
suspend : 중단
end-success: 성공
end-fail: 실패

###
1. 그룹장만 상태 변경 가능
2. suspend 상태에서는 그룹 가입 및 활동 불가
3. end-fail 상태는 그룹 복구 불가
4. suspend 상태에서는 활동 불가
5. 동일 상태로 변경 요청 시 무시(idempotent)

#응답 코드 및 응답 바디
200 OK

|이름   |        타입    |   설명
'groupId'          숫자        그룹 ID
'preState'        문자열      변경 전 상태
'currentState'    문자열      변경 후 상태
'changedAt'       문자열      변경 시각(ISO 8601)

### 응답 예시
{
  'groupId': 1,
  'prevState': "end-success",
  'currentState': "end-fail",
  'changedAt': "2026-05-02T21:00:00Z"
}

### 예외 처리

| 코드 | 상황                      |
| 400 | 잘못된 상태값 요청           |
| 401 | 인증 실패                   |
| 403 | 그룹장이 아님                |
| 404 | 그룹 없음                   |
| 409 | 상태 변경 불가 (이미 end-fail 등) |

### 테스트 케이스
1. state = 'end-success' 인 경우 성공 상태로 변경된다.
2. state = 'progress' 인 경우 진행중 상태로 변경된다.
3. state = 'suspend' 인 경우 활동 중단 상태로 변경된다.
4. state = 'end-fail' 인 경우 삭제 상태로 변경된다.
5. 잘못된 상태값 요청일 경우(state = NULL) 400 에러 반환한다.
6. 그룹장이 아닌 사용자가 요청한 경우 404 에러 반환한다.
7. 존재하지 않는 groupId 인 경우 404 에러
8. 동일 상태 요청인 경우 변경 없이 200 반환한다.
9. 'end-fail' 상태에서 상태 변경을 요청할 경우 409 에러를 반환한다.
