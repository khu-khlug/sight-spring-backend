# Core 레이어 규칙

<!-- CI mixed-change validation test -->

## 역할

- Core는 횡단 관심사를 구현한다.
- 횡단 관심사는 둘 이상의 비즈니스 기능에서 같은 방식으로 적용해야 하는 기능이다. 인증, 인가, 공통 예외 처리, 요청 로그 기록이 횡단 관심사에 해당한다.
- Core는 인증, 인가, 공통 예외 처리처럼 비즈니스 기능과 독립적으로 적용되는 기능을 제공한다.
- Core는 비즈니스 기능의 처리 순서를 결정하지 않는다.

## 구현 규칙

- Core에는 횡단 관심사를 구현하는 클래스만 둔다.
- 인증·인가 코드는 요청자의 식별값과 권한 정보를 제공하거나 확인한다.
- 공통 예외는 HTTP 상태, 사용자에게 보여 줄 메시지, 추가 오류 데이터를 같은 형식으로 표현한다.

## 금지 규칙

- Core는 비즈니스 규칙을 결정하지 않는다.
- Core는 도메인 모델의 상태 변경 규칙과 권한 규칙을 결정하지 않는다.
- Core는 특정 도메인이나 비즈니스 기능에만 필요한 구현을 두지 않는다.

## 의존성 규칙

- Core는 횡단 관심사 구현에 필요한 라이브러리에만 의존한다.
