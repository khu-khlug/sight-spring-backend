# Core 레이어 규칙

## 역할

- Core는 인증, 인가, 공통 예외 처리, 외부 시스템 연결처럼 여러 기능에서 함께 쓰는 기술 코드를 제공한다.
- Core는 외부 시스템과 통신하는 코드가 HTTP 요청 형식, 인증 방식, 오류 응답 형식을 직접 다루도록 한다.
- Core는 업무 처리 순서가 아니라 기술적인 공통 동작만 담당한다.

## 구현 규칙

- Core에는 인증·인가, 공통 예외, 외부 API 클라이언트, 외부 이벤트 수신기, 공통 설정값 접근 코드만 둔다.
- 외부 시스템을 호출하는 코드는 호출에 필요한 인터페이스와 실제 통신 구현을 분리한다.
- 외부 시스템 통신 구현은 주소, 인증 헤더, 요청 형식, 응답 형식, 시간 제한, 통신 오류 처리를 담당한다.
- 인증·인가 코드는 요청자의 식별값과 권한 정보를 제공하거나 확인한다.
- 공통 예외는 HTTP 상태, 사용자에게 보여 줄 메시지, 추가 오류 데이터를 같은 형식으로 표현한다.
- 설정값은 객체를 만들 때 주입받는다.
- 외부 시스템 통신 실패는 호출자가 처리할 수 있는 예외나 결과로 바꾼다.

## 금지 규칙

- Core는 특정 업무 기능의 처리 순서를 결정하지 않는다.
- Core는 업무 데이터의 상태 변경 규칙을 결정하지 않는다.
- Core는 회원, 그룹, 신청서 같은 업무 데이터의 권한 규칙을 결정하지 않는다.
- Core는 Controller, Service, Repository를 호출하지 않는다.
- Core는 API 응답 DTO를 만들지 않는다.
- Core는 HTTP 응답 본문을 만들지 않는다.

## 의존성 규칙

- Core는 Spring Security, HTTP 클라이언트, Discord 클라이언트처럼 공통 기술을 위한 라이브러리에 의존할 수 있다.
- Core는 Controller와 Service의 업무 처리 코드에 의존하지 않는다.
- Service는 Core의 외부 시스템 통신 구현체가 아니라, 그 구현체가 따르는 인터페이스에 의존한다.
